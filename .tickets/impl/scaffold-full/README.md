# Agent-Enriched Scaffolding

## Goal

Add a `--full` flag to `opcom scaffold` that uses an agent session to generate rich Context Packets (Goal, Non-Goals, Constraints, Repo Anchors, Oracle, Tasks) from spec section content instead of the current minimal stubs. Generated tickets are marked with `generated: true` in frontmatter so users know to review them.

## Tasks

- [ ] T1: Add `--full` flag to scaffold CLI command
- [ ] T2: Build agent session setup — pass spec section content, project detection result (stack, services, testing), and existing tickets as context
- [ ] T3: Define Context Packet schema: Goal, Non-Goals, Constraints, Repo Anchors, Oracle, Tasks
- [ ] T4: Feed `TEMPLATE.md` to the agent as a format reference
- [ ] T5: Generate ticket README.md with full Context Packet content and `generated: true` in frontmatter
- [ ] T6: Ensure `--full` composes with `--dry-run` and `--all` flags
- [ ] T7: Without `--full`, behavior remains unchanged (minimal stubs)
- [ ] T8: Tests for agent-enriched scaffold flow

## Acceptance Criteria

- `opcom scaffold docs/spec/foo.md --full` creates tickets with rich Context Packets generated by an agent session
- Generated tickets include `generated: true` in their YAML frontmatter
- `--full --dry-run` previews enriched tickets without writing files
- `--all --full` processes all spec files with agent enrichment
- Without `--full`, existing scaffold behavior is unaffected
